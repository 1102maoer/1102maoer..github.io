MySQL常用命令

查看所有的数据仓库：show databases;

切换要使用的数据仓库：use 数据库名;

推出MySQL: exit;

登录：mysql -u 用户名 -p 密码;

表的设计：

设计原则：建议设计的表尽量遵守三大范式

数据库范式：数据库设计的时候需要遵守的规范，规则。

一般情况下数据库的设计只需要遵循前3范式即可

第一范式：

确保每个字段的原子性，要求表的每个字段必须是不可分隔的独立单元

学生：姓名或者英文名  ----违反第一范式
       路西|lucy
正确做法：
    姓名    英文名

第二范式：

在第一范式的基础上，要求每张表只描述一件事情。表的每个字段都和表的主键有依赖。

表的主键：主键是用来唯一的标识表中一条数据的属性组。

学生和课程
    学号 姓名 年龄 性别 课程编号 课程名称
正确做法：
    学生表：
        学号 姓名 年龄 性别 
    课程表：
        课程编号 课程名称

第三范式：

在第二范式基础，要求每张表的主键之外的其他字段都直接依赖主键，不能有间接的依赖。

学生表：
    学号 姓名 年龄 性别 课程编号
课程表：
    课程编号 课程名称

数据库表的完整性约束：

约束是为了保证表中数据的正确性和完整性.
完整性约束是为了表的数据的正确性！如果数据不正确，那么一开始就不能添加到表中。

六大约束：CONSTRAINT

主键约束：PRIMARY KEY

主键：用来唯一的标识表中一条数据的属性组
主键可以是一个单独的列，也可以是多个列的组合
大部分时候我们是定义一个单独的列作为主键列，列名一般叫XXID

当我们给一个列添加了主键约束，那么这一列的值就必须是非空并且唯一的

CREATE TABLE T_STU(
    SNUM INT PRIMARY KEY
    -- 给学号列添加主键约束 snum列的值将不能为空并且 不能重复
);

非空约束：not null

给一个列添加了非空约束，就是规定这一列的值不能为空

CREATE TABLE T_STU(
    SNUM INT PRIMARY KEY,
    -- 给学号列添加主键约束 snum列的值将不能为空并且 不能重复
    SNAME VARCHAR(25) NOT NULL
    -- 添加非空约束 sname列的值不能为空
);

唯一约束：UNIQUE

给一个列添加唯一约束，就是规定这一列的值唯一不可重复

CREATE TABLE T_STU(
    SNUM INT PRIMARY KEY,
    -- 给学号列添加主键约束 snum列的值将不能为空并且 不能重复
    SNAME VARCHAR(25) NOT NULL,
    -- 添加非空约束 sname列的值不能为空
    PHONE CHAR(11) UNIQUE
    -- 添加唯一约束  phone列的值不能重复，可以为空
);

默认值：DEFAULT

用来在添加一条数据又没有给列指定值的时候，列就使用默认值

CREATE TABLE T_STU(
    SNUM INT PRIMARY KEY,
    -- 给学号列添加主键约束 snum列的值将不能为空并且 不能重复
    SNAME VARCHAR(25) NOT NULL,
    -- 添加非空约束 sname列的值不能为空
    PHONE CHAR(11) UNIQUE
    -- 添加唯一约束  phone列的值不能重复，可以为空
    SEX CHAR(1) DEFAULT '男'
);
sex char(1) default '男' --这是给sex添加默认值
也就是如果添加一条学生数据，没有指定性别，性别默认就是男，如果指定了，指定的是什么就是什么

外键约束：FOREIGN KEY

外键用来表示表与表的关联关系的
外键必须关联到其他表的主键列****
给一个列上添加外键约束，添加的时候会指定外键列关联到是哪张表的那个列
添加了外键约束后，外键列的值必须来源于关联表的关联列，而不能是其他值，但是可以为空

CONSTRAINT 外键约束名字 FOREIGN KEY(外键列列名) REFERENCES 父表名（父表中主键列列名）；

如果父表中的数据删除时，子表中的数据处理：

默认是子表中有数据关联父表时，父表中的数据不能删除
常用的设置：

NO ACTION :

当删除父表数据时，子表中没有动作
有关数据的时候父表中的数据删除不掉

SET NULL:

当删除父表数据的时候，子表中关联列的值设置为空

CASCADE:

设置级联删除
当删除父表数据的时候，子表中关联的数据也同时删除

检查约束—-MySQL不支持

check(age>6 and age<20)

主键补充—主键自增：

主键约束—保证主键列的值非空并且唯一

当我们往表中插入数据的时候，保证主键值不能重复还是有点麻烦的

mysql数据库中提供了主键自增功能

当添加一条数据的时候，可以不用指定主键的值，mysql数据库会自动的生成主键的值，生成的值其实就是原来的主键最大值+1

建表的时候可以在添加主键约束的时候指定主键值自动增长

列名 数据类型 primary key AUTO_INCREMENT;

注意：主键列的类型必须是整数型才能自增
oracle数据库是没有主键自增，保证主键值不重复---序列

表和表之间的关联关系：

一对一：人和身份证
一个人只会对应一张身份证
一个身份证只对应一个人

​ 表示一对一的关系时，可以在两张表的任意的表中设置外键

​ 一对一的表设计：
​ 如果内容基本上都是需要操作，也可以将两表合成一张表
​ 如果说经常需要用到某一项的单独信息，建议分成两张表，两张表之间可以使用外键关联

一对多/多对一：
学生和班级
一个学生只能对应一个班级
一个班级可以有多个学生

​ 一对多的表设计：
​ 一般是在多方添加外键，关联到一方的主键

    学生表：
​        学号 姓名 年龄 班号

​    班级表：
​        班号 班级名称

​

多对多：
学生和课程
一个学生可以学习多门课程
一门课程可以有多个学生学习

​ 多对多关系的表设计：使用一个中间表

​ 

学生表：
        学号 姓名 年龄 
        1001
        1002
        1003
    课程表：
        课程编号 课程编程
        001
        002
    中间表 学生-课程:
        stu_course
        学号 课程号 成绩
        1001  001
        1001  002
        1002  002
        1002  003

SQL语句

登录数据库服务器后，要操作数据库中的数据，要使用SQL语句。

SQL: SQL（Structured Query Language）是“结构化查询语言”，它是对关系型数据库的操作语言。它可以应用到所有关系型数据库中，例如：MySQL、Oracle、SQL Server等。

语法要求:

SQL语句可以单行或多行书写，以分号结尾；
可以用空格和缩进来来增强语句的可读性；
关键字不区别大小写，建议使用大写；

SQL分类：

DDL

（Data Definition Language）：数据定义语言，用来定义数据库对象：库、表、列等；

主要用在建表 删表 建库 删库

create……

drop……

创建数据仓库：CREATE DATABASE 数据库名 DEFAULT CHARACTER SET 字符集；

修改数据库编码：ALTER DATABASE 数据库名 CHARACTER SET utf8;

切换数据仓库使用： USE 数据库名；

删除数据仓库：DROP DATABASE 数据库名；

CREATE DATABASE TEXT DEFAULT CHARACTER SET utf8;-- 创建

SHOW DABABASES;-- 展示数据仓库

USE TEXT;-- 使用

SHOW TABLES;-- 展示表

DROP DATABASE TEXT;-- 删除数据仓库

对表结构进行操作：

创建表： CREATE TABLE 表名（-- 在小括号中指定表中有哪些列以及每个列的类型

​                    列名 数据类型  约束

​             ）;
表名和列名的命名规则：和java的标识符的命名规则相似

数据类型：常用的：
整型：int
字符串：
char:固定长度字符串类型。char(4)，当数据长度为2时，会在后面添加2个空格补位。
varchar:固定长度字符串类型。char(4)，当数据长度为2时，会在后面添加2个空格补位。
一般对于长度固定的字段使用char,比如性别，手机号，身份证号
一般长度不定的字符串字段使用varchar,比如姓名，地址。。。

浮点数：double decimal

布尔型:boolean

日期：
date yyyy-mm-dd
datetim

修改表结构：

添加字段：
ALTER TABLE 表名 ADD COLUMN 列名 数据类型；
删除字段：
ALTER TABLE 表名 DROP COLUMN 列名；
修改字段类型：
ALTER TABLE 表名 MODIFY COLUMN 列名 数据类型；
修改字段名称：
ALTER TABLE 表名 CHANGE COLUMN 原列名 新列名 数据类型；
修改表名称：
ALTER TABLE 表名 RENAME TO 新的表名；
删除表：
DROP TABLOR 表名；
删除表结构的时候表中的数据同时删除。

DML

（Data Manipulation Language）：数据操作语言，用来定义数据库记录（数据）；

主要用在对表中数据的增删改操作

insert….

插入数据：

给指定列指定值：
INSERT INTO 表名（列名1，列名2...）VALUES (值1，值2)；
向表中添加一条数据
给表中所有的列都指定值：
INSERT INTO 表名 VALUES (值1，值2...)
注意：这种写法的值的顺序和表中列的顺序是一一对应的

update….

修改表中指定行的指定列的值

UPDATE 表名 SET 列名1=值1，列名2=值2 WHERE 条件

where后面的条件是过滤条件，过滤的是行
也就是通过where后面的条件控制要更新哪些行数据的指定列的值
如果没有where条件，就是更新表中所有的行的指定列的值
update语句使用的时候一般都是有过滤条件的

delete…..

删除表中指定的行

DELETE FROM 表名 WHERE 条件

where后面的条件是过滤条件，过滤的是行
只删除满足过滤条件的行
如果没有where条件，就是删除表中所有的数据
一般情况下delete语句都是有过滤条件的

删除表中所有的数据：
delete from 表名;

–是直接将表中所有的数据删除，可以回滚事务先把数据放到“回收站”
truncate table 表名;–删除表中所有的数据
–其实是先将整张表删除 然后有重新创建了一张新的表不会把数据放到“回收站” 不会回滚

DQL

（Data Query Language）：数据查询语言，用来查询记录（数据）。

主要用在对表中数据的查询操作

select….

DCL

（Data Control Language）：数据控制语言，用来定义访问权限和安全级别；

授权：grant…

撤销权限：revoke…
